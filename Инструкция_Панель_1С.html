1. Что это такое

Это веб-приложение (Google Apps Script), привязанное к вашей Google Таблице «Отчёт». Оно автоматически:

  • сопоставляет товары электротранспорта между разными системами (1С, поставщики, таблица поставок) по названию, даже когда названия написаны по-разному;
  • подтягивает остатки, продажи, статус поставок и предстоящих отгрузок прямо в таблицу;
  • показывает всё это на одной странице в виде дашборда;
  • может обновляться как вручную (кнопкой), так и автоматически по расписанию (11:00 и 16:00 ежедневно).

2. Первоначальная установка (один раз)

2.1. Вставить код

  1. Откройте таблицу «Отчёт» → меню Расширения → Apps Script.
  2. Файл Код.gs — удалите всё содержимое, вставьте код из файла Code.gs.txt.
  3. Файл UploadStock (HTML) — если его нет, создайте: значок + рядом с «Файлы» → HTML → назовите ровно UploadStock. Вставьте код из файла UploadStock.html.
  4. Сохраните проект (значок дискеты).

2.2. Включить сервис Drive API

  1. Слева в редакторе — раздел «Сервисы» → значок +.
  2. Найдите Drive API → добавьте, обязательно выбрав версию v2 в выпадающем списке.
Критично: без этого шага загрузка отчётов (.xls/.xlsx/.htm) не будет работать — появится ошибка о недостающей авторизации или «Drive is not defined».

2.3. Развернуть как веб-приложение

  1. Кнопка «Начать развёртывание» → «Новое развёртывание».
  2. Тип: Веб-приложение.
  3. Выполнять как: Пользователь, обращающийся к веб-приложению.
  4. У кого есть доступ: Все, у кого есть аккаунт Google.
  5. Развернуть → пройти авторизацию (Проверить разрешения → аккаунт → Дополнительно → Перейти к проекту → Разрешить).
  6. Скопируйте выданную ссылку вида https://script.google.com/macros/s/.../exec — сохраните в закладки, это и есть ваша рабочая страница.
Не забывайте: если позже меняете код (Код.gs или UploadStock) — нужно каждый раз делать новое развёртывание (Управление развёртываниями → ✏️ → Версия → Новая версия → Развернуть), иначе по старой ссылке будет открываться старый код.

3. Дашборд на странице

Сверху — сводная статистика (позиций в каталоге, штук на складах, сколько в пути, сколько готовится к отгрузке, брак).

Ниже — вкладки по листам таблицы: «Остатки», «Статистика продаж», «История остатков». Переключаются кликом, показывают актуальные данные.

Индикатор связь есть · обновлено ЧЧ:ММ:СС в шапке коротко вспыхивает синим при каждом успешном обновлении — если он покраснел и пишет нет связи с таблицей, значит запрос к Google Таблице не прошёл (проверьте развёртывание/авторизацию).

Обратите внимание: данные на странице обновляются при открытии/перезагрузке страницы (F5) и кнопкой «Обновить данные» под таблицей — сами по себе, без вашего участия, они не обновляются, пока страница открыта.

4. Пять действий (кнопки)

01 Остатки на складах

Источник: отчёт «Остатки ТМЦ на складах», выгруженный из 1С (.xls/.xlsx/.htm).

Складывает количество по складам Матусевича + Бобруйск + Дзержинск → столбец E «Остаток на текущий день». Брак → столбец F. Переучёт → столбец G.

Дополнительно: каждый раз сохраняет снимок остатков в скрытый лист «История остатков» и пересчитывает столбцы I («15 дней назад») и J («30 дней назад») — данные там появятся, когда накопится история за этот срок.

02 Продажи за период

Источник: «Ведомость по остаткам ТМЦ» за нужный период (столбец «Расход»).

Перед загрузкой вводится название периода (например «Июнь»). Записывает продажи в лист «Статистика продаж»: отдельный столбец на каждый период, столбец B «Итог 2026» — автосумма формулой по всем периодам.

Важно: если загружаете кумулятивный отчёт с начала года по июнь — назовите период «Июнь» (это будет как бы «итог до июня»). Следующий раз для июля выгружайте из 1С период только за июль, не с начала года — иначе продажи задвоятся.
03 Входящая поставка (МЦ-Аомори)

Источник: внешняя таблица «Поставки», лист с товарными позициями по контейнерам.

Сканирует все строки за всё время. Для каждого товара берётся его собственная самая свежая по дате запись (если товар встречается в нескольких поставках — побеждает более поздняя дата). Учитываются только позиции, написанные чёрным цветом текста внутри ячейки.

Результат → столбец H «Пришло (последняя поставка)» листа «Остатки».

04 Уже в пути

Тот же источник, что и «Входящая поставка», но по-другому: для товаров, которые ещё не отмечены в столбце «НА СКЛАДЕ», берётся самый поздний известный этап доставки:

  • Столбец N «Дата прихода» — если заполнен, приоритет №1;
  • Столбец M «Оформлен» — приоритет №2;
  • Столбец L «Прибыл ПТО» — приоритет №3;
  • Столбец K «Прибытие орентир.» — приоритет №4;
  • Столбец I «Статус» — если больше ничего нет.

Результат → столбец K листа «Остатки», в формате «дата (этап) — N шт». Как только по этой поставке появляется дата в столбце «НА СКЛАДЕ» — запись автоматически убирается (товар больше не «в пути», а на складе).

05 Готовятся к отгрузке

Источник: другая таблица — заказы у поставщиков («Скутеры»). Название товара берётся из столбцов D (модель) + E (акб), количество — из столбца F «штук».

Учитываются только строки с датой в столбце B от 1 июня текущего года и по сегодняшний день. Если в столбце G уже есть любая отметка (отгружено / номер рейса) — строка исключается, товар для этого потока уже не «готовится», а в пути.

Результат → столбец L листа «Остатки», в формате «дата — N шт».

5. Автоматическое обновление по расписанию

Три действия — «Входящая поставка», «Уже в пути», «Готовятся к отгрузке» — можно обновлять автоматически, без открытия страницы.

  1. Раскройте на странице «Отладка и служебные инструменты».
  2. Нажмите «Включить автообновление» (один раз).
  3. Нажмите «Проверить статус» — должно показать «Активно, триггеров установлено: 2».

После этого каждый день примерно в 11:00 и примерно в 16:00 (Google не гарантирует точную минуту, обычно в пределах часа) скрипт сам прогонит все три шага по очереди. Результат последнего автозапуска виден прямо под шапкой страницы: дата/время и отметка ✓/✗ по каждому шагу.

Надёжность: один упавший шаг не останавливает остальные — каждый выполняется независимо.

6. Устранение типичных неполадок

СимптомПричинаРешение
Кнопка ничего не делает, статус не меняется JS-код обрезался при копировании, или не сделано новое развёртывание Вставить файл заново целиком (проверить начало и конец), сохранить, сделать новое развёртывание
«Ошибка: Для выполнения этого действия необходима авторизация» Скрипту не выданы все нужные права (особенно на Google Диск) В редакторе выбрать функцию (например любую из process...) → Выполнить → пройти авторизацию полностью
«ReferenceError: Drive is not defined» Сервис Drive API не подключён, или подключена версия v3 вместо v2 Сервисы → добавить Drive API, версия v2. Если уже есть — удалить и добавить заново
Страница открывается только в режиме инкогнито В обычном браузере залогинено несколько Google-аккаунтов одновременно, путается сессия В развёртывании: «Выполнять как» → Пользователь, обращающийся к веб-приложению; «Доступ» → Все, у кого есть аккаунт Google
После правки кода на странице всё по-старому Не сделано новое развёртывание — ссылка держит старую версию Управление развёртываниями → ✏️ → Версия → Новая версия → Развернуть
Много «не найдено» при загрузке остатков/продаж Опечатка в названии в столбце C, либо у товара реально не было движения за период Сверить конкретные позиции вручную; отсутствие движения — это не ошибка
