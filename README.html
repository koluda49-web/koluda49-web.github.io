OK-CRM → Google Drive бот: инструкция по установке и использованию

Бот автоматически обрабатывает замороженные заказы в рассрочку: генерирует платежи, скачивает документ, раскладывает его по нужным папкам на Google Drive, вставляет ссылку в заказ и переводит его в статус «Новый».

Что делает бот, по шагам

Для каждого подходящего заказа (статус «Заморожен», банк «Фирма рассрочка», метод оплаты «Расчетный счет»):

  1. Читает цену, % первоначального взноса и текст пометки к заказу.
  2. Определяет сумму первого взноса — если в пометке явно написана сумма («первоначальный взнос 1000», «предоплата 500», «аванс 300» и т.п. вариации) — берёт её; если нет — считает по проценту от цены. Если не удалось разобрать пометку однозначно — заказ пропускается и уходит в manual_review.csv для ручной проверки, а не обрабатывается вслепую.
  3. Вписывает сумму в поле «1-ый взнос», жмёт «Генерировать платежи» (если платежи для заказа уже есть — этот шаг пропускается, сайт сам блокирует повторную генерацию).
  4. Через шестерёнку жмёт «Генерировать документы», скачивает получившийся .zip, распаковывает из него .docx.
  5. На Google Drive заходит в 2026 Рассрочки → <месяц> → <дата оформления>, создаёт там папку <id заказа> <фамилия клиента> (если её ещё нет), загружает туда .docx.
  6. Открывает редактирование заказа, дописывает в «Пометку к заказу» новую строку со ссылкой на созданную папку и строкой «сделано!!», меняет статус на «Новый», сохраняет.

Файлы проекта

run_bot.bat              - двойной клик, чтобы просто запустить бота
bot.py                   - основной скрипт
config.py                - настройки (адрес CRM, ID папки Диска, пороги и т.п.)
selectors.py              - селекторы элементов страницы CRM
drive_browser.py          - управление Google Drive через браузер
parse_notes.py             - разбор текста пометки на сумму взноса
state_store.py             - локальная база учёта уже обработанных заказов
storage_state.json         - СЕССИЯ ВХОДА (самый важный файл, см. ниже)
requirements.txt            - список Python-зависимостей
grab_session_from_cdp.py     - служебный скрипт захвата сессии (см. "Если сессия слетела")
manual_review.csv, errors.csv - журналы для проверки человеком (создаются автоматически)
state.sqlite                - база "уже обработанных" заказов (создаётся автоматически)

Тестовые/вспомогательные скрипты (test_*.py, import_cookies.py, convert_cookies.py, auth_login.py) использовались только на этапе настройки, для повседневной работы не нужны, но можно оставить в папке - не мешают.

Повседневное использование (когда всё уже настроено)

Просто дважды кликните на run_bot.bat. Откроется чёрное окно консоли, бот сам активирует окружение и начнёт обработку. Браузер при этом не открывается на экране (работает в фоне) — это нормально, так и задумано, чтобы не мешать вам работать за компьютером.

По завершении в окне появится строка вида:

Готово. Обработано за этот прогон: N

и окно будет ждать нажатия любой клавиши, чтобы вы успели прочитать результат.

Если заказов, подходящих под фильтр, больше 20 (одна страница списка в CRM) — бот обработает только первые 20 за один запуск. Обработанные заказы меняют статус и перестают попадать под фильтр «Заморожен», поэтому просто запустите run_bot.bat ещё раз — на первой странице к этому моменту уже будут следующие заказы. Повторяйте, пока не увидите «Обработано за этот прогон: 0».

После каждого прогона стоит заглянуть в: - manual_review.csv — заказы, где сумму взноса не удалось однозначно определить по тексту пометки (пустая пометка, нестандартная формулировка, или сумма из текста сильно разошлась с расчётом по проценту). Их нужно обработать вручную или поправить пометку и запустить бота снова. - errors.csv — заказы, на которых бот столкнулся с технической ошибкой (например, не удалось найти какой-то элемент на странице). Тоже требуют ручного разбора.

Первоначальная настройка на новом компьютере

1. Скопировать проект

Скопируйте всю папку с ботом на новый компьютер, кроме подпапки venv (она пересоздаётся заново на каждом компьютере отдельно, не переносится). Остальное - все .py/.bat/.txt файлы и, что особенно важно, storage_state.json - переносите как есть.

2. Установить Python

Проверьте в cmd:

python --version

Если нет - установите с python.org, обязательно поставив галочку Add Python to PATH при установке.

3. Создать окружение и поставить зависимости

В папке проекта:

python -m venv venv
venv\Scripts\activate.bat
pip install -r requirements.txt
playwright install chromium

4. Проверить, что сессия работает

python bot.py --dry-run --limit 1

Если видите вывод вида [ID] сумма взноса = ... (источник: ...) - всё работает, сессия перенеслась успешно. Если возникает ошибка входа - см. раздел ниже.

5. Дальше - обычное использование

Просто run_bot.bat, как описано выше.

Если сессия входа "слетела" (протухла)

Google-сессии не живут вечно. Если бот вдруг перестал заходить в CRM или на Диск (в логе видны ошибки входа/редиректы на страницу логина Google) - нужно заново захватить сессию:

  1. Полностью закройте все окна Chrome.
  2. Откройте новое окно cmd и выполните (путь может отличаться, если Chrome установлен не в стандартную папку):
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:\chrome-remote-profile"
  1. В открывшемся окне Chrome зайдите на a.ok-crm.com и на drive.google.com, залогиньтесь через Google-аккаунт как обычно в обоих.
  2. Не закрывая это окно, в другом окне cmd (в папке проекта, с активным venv):
python grab_session_from_cdp.py
  1. Должно вывестись Успех: сессия (cookies + localStorage) сохранена в storage_state.json. Окно Chrome с отладочным портом после этого можно закрыть.
  2. Проверьте: python bot.py --dry-run --limit 1.

Полезные флаги запуска (для отладки, не для повседневного использования)

python bot.py --dry-run              REM ничего не менять, только показать план
python bot.py --dry-run --limit 1    REM то же самое, но только для одного заказа
python bot.py --order-id 593570      REM обработать только конкретный заказ по ID
python bot.py --headed               REM показать окно браузера вживую (без ускоренного режима)
python bot.py --headed --pause       REM показать браузер и делать паузу после каждого заказа - для пошагового наблюдения

Обычный run_bot.bat эквивалентен запуску python bot.py без флагов - боевой режим, без окна браузера, без пауз.

Известные особенности и ограничения

  • Разбор пометки к заказу: список распознаваемых формулировок суммы взноса можно расширять в parse_notes.py (переменная _PATTERNS), если появятся новые варианты, которые бот не распознаёт - такие заказы будут попадать в manual_review.csv, это ожидаемо и безопасно (лучше пропустить и попросить проверить вручную, чем угадать неверно).
  • Google Drive использует обычный веб-интерфейс (не API) - тем же браузером и той же сессией, что и CRM. Если Google обновит вёрстку своего интерфейса, некоторые селекторы в drive_browser.py могут перестать работать - тогда потребуется заново снять актуальные селекторы (см. историю настройки, метод через playwright codegen).
  • Пагинация списка заказов не используется - бот всегда работает только с первой страницей списка и рассчитывает на то, что обработанные заказы сами "уходят" из фильтра «Заморожен», освобождая место для следующих. Если фильтр в CRM когда-либо изменится так, что обработанные заказы перестанут выпадать из выборки - потребуется пересмотреть эту логику.
