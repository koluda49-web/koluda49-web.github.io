1. Ежедневный запуск (обычный режим)

Для повседневной работы просто дважды кликните файл Запустить бота.bat в папке проекта. Откроется окно консоли, бот сам:

  • найдёт на текущей странице заказы со статусом «Заморожен» (банк «Фирма рассрочка», метод оплаты «Расчетный счет»);
  • обработает каждый по очереди — впишет сумму взноса, сгенерирует документы, загрузит их на Google Drive, сменит статус на «Новый»;
  • заказы, обработка которых не удалась, запишет в errors.csv, manual_review.csv или skipped_orders.csv — с этими файлами стоит сверяться после каждого прогона.
Если заказов больше, чем на одной странице Просто запустите Запустить бота.bat ещё раз — обработанные заказы меняют статус и выпадают из фильтра «Заморожен», так что следующий прогон автоматически возьмётся за следующую пачку.

2. Если сессия Google Drive протухла

Бот теперь сам проверяет сессию Drive в самом начале прогона и, если она не годится, сразу останавливается с инструкцией — не дожидаясь, пока упадёт где-то в середине обработки заказа. Если это произошло, повторите вход:

1
Откройте обычное окно cmd и запустите Chrome с отладочным портом: 
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="C:\chrome-remote-profile"
2
В открывшемся окне Chrome зайдите на a.ok-crm.com и убедитесь, что видите список заказов (не форму входа).
3
В новой вкладке того же окна зайдите на drive.google.com — убедитесь, что видите обычный залогиненный интерфейс с левой панелью «Мой диск», «Компьютеры» и т.п., а не гостевой просмотр без бокового меню. Если видите форму входа или надпись «Вы вышли из аккаунта» — войдите заново.
4
Не закрывая это окно Chrome, откройте второе окно cmd и выполните: 
cd C:\okcrm_bot
venv\Scripts\activate.bat
python grab_session_from_cdp.py
Должно вывести «Успех: сессия (cookies + localStorage) сохранена в storage_state.json».
5
Теперь оба окна Chrome/cmd с отладочным портом можно закрыть. Запускайте бота как обычно.
Почему это вообще происходит Google периодически завершает сессию по соображениям безопасности — это никак не убрать полностью, только повторять этот вход по мере надобности. Хранить пароль в файле и подставлять его автоматически не решение — Google сам блокирует именно автоматизированный вход по логину/паролю, поэтому и используется способ через ручной вход + захват cookies.

3. Проверка / переобработка одного заказа

ФлагЧто делает
--dry-runНичего не сохраняет и не загружает — только показывает, как бот прочитал данные и посчитал сумму взноса.
--order-id 12345Обработать только один конкретный заказ по ID, минуя сбор общего списка.
--headedПоказать окно браузера (по умолчанию бот работает в фоне, без видимого окна).
--pauseОстановиться и ждать Enter после каждого заказа — удобно для ручной сверки.
--limit NОбработать не больше N заказов за один прогон.

Пример безопасной проверки одного заказа перед боевым прогоном:

python bot.py --dry-run --order-id 597202

Пример полного прогона одного заказа с показом браузера и паузой для сверки:

python bot.py --order-id 597202 --headed --pause

Также есть готовый файл Запустить бота (с проверкой).bat — то же самое, что обычный запуск, но с паузой после каждого заказа. Удобно использовать разово, когда хочется вручную сверять каждый результат.

4. Если заказ вручную вернули на «Заморожен»

Бот запоминает уже обработанные заказы в state.sqlite и по умолчанию их пропускает — это защита от случайного повторного запуска. Если вы сознательно откатили заказ обратно (например, потому что сумма взноса была неверной, и вы вручную удалили платежи/статус) — используйте флаг --force вместе с --order-id:

python bot.py --order-id 596447 --force
Важно --force работает только вместе с --order-id — на обычные пакетные прогоны без указания конкретного заказа он не влияет, там защита state.sqlite остаётся как есть.
Дубли документов на Диске При каждом повторном запуске с --force документ заново загружается на Google Drive — старая копия не удаляется автоматически. Если заказ обрабатывался несколько раз, зайдите в его папку на Диске и удалите лишние дубли вручную.

5. Что делать при ошибках

ФайлЧто туда попадает
manual_review.csvЗаказы, где сумму взноса не удалось однозначно определить из текста пометки — нужно проверить вручную и решить сумму самостоятельно.
errors.csvТехнические сбои бота (таймауты, неожиданные ошибки) — заказ не тронут, можно смотреть текст ошибки и путь к скриншоту (столбец с текстом ошибки).
skipped_orders.csvЗаказы, которые сама CRM не даёт обработать (например, баннер «Проверьте ФИО клиента!») — это не ошибка бота, а проблема данных самого заказа.
downloads\error_screenshots\Скриншот страницы в момент любой ошибки — называется вида ID_типОшибки_дата_время.png. Всегда смотрите последний скриншот, если что-то пошло не так — обычно сразу видно причину.

6. Что за файлы в проекте

ФайлНазначение
bot.pyГлавный скрипт — вся логика обработки заказов.
config.pyНастройки: адрес CRM, ID папки на Диске, пороги, пути к файлам.
selectors.pyСелекторы для поиска элементов на странице CRM.
parse_notes.pyРазбор текста «Пометки к заказу» на сумму первого взноса.
drive_browser.pyРабота с Google Drive через обычный веб-интерфейс (создание папок, загрузка файлов).
state_store.pyЛокальная база (SQLite) для идемпотентности + CSV-логи ошибок.
grab_session_from_cdp.pyЗахват сессии (cookies) из вручную залогиненного Chrome.
storage_state.jsonСохранённая сессия входа — используется ботом при каждом запуске.
state.sqliteИстория уже обработанных заказов.
Запустить бота.batОбычный запуск без пауз, для повседневной работы.
Запустить бота (с проверкой).batЗапуск с паузой после каждого заказа — для ручной сверки.
Файлы, которые больше не используются drive_utils.py, auth_login.py, import_cookies.py, convert_cookies.py — более ранние подходы, которые не прижились (Google блокировал автоматизированный OAuth-вход, а cookies через browser_cookie3 не расшифровывались из-за защиты новых версий Chrome). Оставлены как справка, можно удалить при уборке проекта.

7. Перенос проекта на другой компьютер

Проект переносится целиком одной папкой, но окружение Python (папку venv) переносить не нужно — её лучше пересоздать заново на новом компьютере, так как она содержит абсолютные пути к старому компьютеру и может не заработать на другом.

Что скопировать

  • Всю папку проекта (C:\okcrm_bot) — все .py-файлы, config.py, selectors.py, requirements.txt, оба .bat-файла;
  • state.sqlite — если хотите сохранить историю уже обработанных заказов (чтобы бот на новом компьютере не начал их обрабатывать заново);
  • storage_state.json — можно скопировать для пробы, но, скорее всего, сессию всё равно придётся захватывать заново на новом компьютере (см. ниже).

Что НЕ копировать

  • venv\ — пересоздаётся заново на месте;
  • __pycache__\ — временный кэш Python, не нужен;
  • downloads\ — временные скачанные файлы, безопасно не переносить.

Шаги на новом компьютере

1
Установите Python (при установке отметьте галочку «Add python.exe to PATH») и Google Chrome, если их ещё нет.
2
Скопируйте папку проекта на новый компьютер, например в C:\okcrm_bot.
3
Откройте cmd, перейдите в папку и создайте окружение заново: 
cd C:\okcrm_bot
python -m venv venv
venv\Scripts\activate.bat
pip install -r requirements.txt
playwright install chromium
4
Захватите сессию заново на этом компьютере — даже если storage_state.json был скопирован со старого, сессия Google почти наверняка не будет работать с нового устройства/IP (это тот же механизм защиты Google, что описан в разделе 2). Повторите шаги из раздела «Если сессия Google Drive протухла» — то есть заново: Chrome с отладочным портом → вход на CRM и Drive → python grab_session_from_cdp.py.
5
Проверьте всё так же, как в самый первый раз: 
python test_session.py
python test_drive_session.py
python bot.py --dry-run --order-id <любой ID заказа>
Важно про безопасность storage_state.json содержит действующие cookies входа — это по сути ключ доступа к аккаунту Google и к CRM. Не отправляйте этот файл через мессенджеры/почту в открытом виде и не оставляйте в местах с общим доступом — тот, у кого окажется этот файл, сможет действовать от имени вашего аккаунта, пока сессия не истечёт.
